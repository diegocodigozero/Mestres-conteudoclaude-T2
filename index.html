Tópico 02

Claude Desktop vs Claude Terminal (CLI)

Antes de instalar nada, vale entender a diferença entre os dois jeitos de usar o Claude, e por que vamos acabar usando os dois ao longo do curso.

Claude Desktop

Ecossistema completo

Tudo numa interface visual integrada. Ideal para conversar, criar e demonstrar.

  • Claude Chat
  • Claude Cowork
  • Claude Code
  • Artefatos
  • Projetos

Claude Terminal (CLI)

Poder bruto via linha de comando

Linha de comando focada em desenvolvimento. Roda em qualquer ambiente, inclusive remoto.

  • Claude Code
?

Mas o Desktop já tem o Code, não é a mesma coisa?

Não. Mesmo sendo o mesmo "Claude Code", a forma de usar e o que cada um permite muda.

✓ Vantagens do CLI

  • Mais rápido no dia a dia
  • Funciona em terminal remoto (servidor, máquina de outro lugar)

✗ Desvantagens

  • Pode assustar um pouco quem nunca usou terminal
?

Qual a diferença entre o Code do Desktop e o Code do CLI?

A diferença real está em como você interage e o que cada um permite fazer:

  • Desktop: passa do plan mode para a execução automaticamente.
  • CLI: você precisa acionar manualmente a execução. Em compensação, sessões paralelas são bem mais fáceis.
Resumindo

Use os dois. O CLI pra refatoração pesada e builds noturnos. O Desktop quando quiser revisar mudanças visualmente ou demonstrar pra alguém não-técnico.

Desktop é melhor pra interação visual.
CLI é melhor pra automação e poder bruto.

Tópico 03

Instalação do Git e Node.js

Antes de qualquer coisa, precisamos preparar o ambiente. Vamos instalar duas ferramentas essenciais: o Git (para controle de versão) e o Node.js (runtime JavaScript usado pelo Claude Code e por boa parte das ferramentas modernas de dev).

1. Git

Controle de versão

O Git é o padrão da indústria para versionar código. Vamos usá-lo para acompanhar mudanças, criar branches e (depois) publicar no GitHub.

Endereço de download https://git-scm.com/install/windows

Validação

Quando terminar, abra um novo PowerShell e valide:

git --version

Deve aparecer algo como git version 2.47.x.

2. Node.js

Runtime JavaScript

O Node.js permite rodar JavaScript fora do navegador. É a base de praticamente todas as ferramentas que vamos usar, incluindo o próprio Claude Code.

Endereço de download https://nodejs.org/en/download

Validação

Abra um novo PowerShell e valide:

node --version
npm --version

Devem aparecer as versões instaladas (ex.: v24.x.x e 10.x.x).

i
Importante: sempre abra um novo terminal após instalar uma ferramenta. As variáveis de ambiente só carregam em terminais novos.

Tópico 08

Modelos e nível de esforço

O Claude tem diferentes modelos, cada um com um equilíbrio entre velocidade e profundidade de raciocínio. Você escolhe o modelo certo pra cada tarefa.

Modelos disponíveis

1

Opus 4.7

Mais potente

O modelo top de linha. Pensa fundo, conecta muitos pontos e dá respostas bem trabalhadas.

Use em: análises complexas, código difícil, planejamento.
2

Opus 4.7 1M

Memória gigante

Igual ao Opus 4.7, mas com janela de 1 milhão de tokens. Lê projetos inteiros sem perder o fio.

Use em: bases de código grandes, livros, longas conversas.
3

Sonnet 4.6

Equilibrado

O meio-termo: rápido o suficiente pro dia a dia e inteligente o suficiente pra quase tudo.

Use em: tarefas do dia a dia, chats normais.
4

Haiku 4.5

Mais rápido

Modelo leve e veloz. Responde quase instantâneo, ideal pra coisas simples.

Use em: perguntas rápidas, resumos curtos, classificações.
5

Opus 4.6 Legado

Versão antiga

Versão anterior do Opus. Continua disponível pra quem quer comparar ou já tem fluxos rodando nela.

Use só se: precisar reproduzir resultados antigos.

Nível de esforço

Diz pro modelo o quanto ele deve "pensar" antes de responder. Quanto maior o esforço, mais cuidadosa é a resposta, mas também demora mais.

Baixa Resposta direta, quase sem raciocínio extra.
Média Pensa um pouco antes de responder.
Alta Análise mais cuidadosa, considera alternativas.
Extra alto Raciocínio bem profundo. Padrão recomendado.
Max Máximo possível. Mais lento, melhor qualidade.
i
Modo rápido: ativa quando você quer resposta imediata, sem o modelo "pensar" antes. Útil pra perguntas triviais e diálogos curtos.
Atalhos no Desktop: Ctrl + I abre os modelos · Ctrl + E abre o esforço
Tópico 09

Como economizar tokens

Cada mensagem consome um pedaço do seu limite. Com hábitos simples, você estica seu uso, recebe respostas mais úteis e ainda fica menos tempo esperando.

Hábitos que economizam tokens

1

Conversa nova ao mudar de assunto

O modelo lê todo o histórico a cada resposta. Se você troca de tema, o contexto antigo só atrapalha e gasta tokens à toa.

Ex.: terminou de discutir um bug? Abre nova conversa para falar sobre design.
2

Seja específico no pedido

Pergunta vaga vira resposta inflada. Diga o quê, por quê e como você quer o formato.

Ruim: "fala sobre useEffect". Bom: "explique em 3 bullets quando NÃO usar useEffect em React 19."
3

Escolha o modelo certo pra tarefa

Para coisas simples (resumir, formatar, traduzir, código trivial), use Haiku ou Sonnet. Reserve o Opus pra problemas que pedem raciocínio profundo.

Tradução curta = Haiku. Refatorar arquitetura = Opus.
4

Ajuste modelo e esforço por mensagem

No chat você troca o modelo e o nível de esforço a qualquer momento (Ctrl+I e Ctrl+E). Não precisa ficar no top de linha o tempo todo.

Pergunta rápida? Baixa o esforço pra "Baixa" e usa Haiku.
5

Use /compact em conversas longas

Quando o histórico fica gigante, o comando /compact resume a conversa até ali, libera contexto e mantém o essencial.

Conversa de 50+ trocas? Roda /compact e segue do ponto onde parou.
6

Skill /caveman pra respostas ultra-curtas

Modo "caveman" faz o Claude responder em estilo telegráfico, cortando ~75% dos tokens sem perder a informação técnica.

GitHub Ideal pra debug, listas e checklists técnicos.

Tópico 11

Navegando pelo terminal

Antes de chamar o Claude, você precisa estar dentro da pasta certa. Aqui vão os comandos essenciais pra andar pelos diretórios, criar pastas e os principais atalhos do Claude Code.

1. Saber onde você está e o que tem

Dois comandos resolvem 90% do trabalho de navegação: pwd mostra a pasta atual, ls lista o que tem dentro dela.

PowerShell
PS C:\Users\Voce> pwd
C:\Users\Voce
PS C:\Users\Voce> ls
Mode Name
---- ----
d---- Desktop
d---- Documents
d---- Downloads
-a--- anotacoes.txt
PS C:\Users\Voce>

2. Entrar e sair de pastas com cd

cd (change directory) é o "andar" pelo computador. Use cd .. pra voltar uma pasta.

PowerShell
PS C:\Users\Voce> cd Documents
PS C:\Users\Voce\Documents> cd ..
PS C:\Users\Voce>

3. Criando a pasta do projeto

mkdir (make directory) cria uma pasta nova. Depois é só entrar nela e rodar o Claude lá dentro.

PowerShell 
mkdir meu-projeto
cd meu-projeto
claude
PS C:\Users\Voce> mkdir meu-projeto
Directory: C:\Users\Voce\meu-projeto
PS C:\Users\Voce> cd meu-projeto
PS C:\Users\Voce\meu-projeto> claude
✻ Welcome to Claude Code
Type your message or /help for commands
>

4. Principais slash commands do Claude

Dentro do Claude Code, qualquer comando que começa com / é uma ação rápida. Digite /help pra ver a lista completa.

Claude Code
> /help
Comandos disponíveis:
/help Lista todos os comandos
/clear Limpa a sessão atual
/compact Resume a conversa e libera contexto
/model Troca o modelo (Opus, Sonnet, Haiku)
/init Cria o arquivo CLAUDE.md no projeto
/review Revisa o código atual
/status Mostra status da sessão
/cost Mostra o custo até agora
/permissions Gerencia permissões de ferramentas
/login Autentica com a conta Claude
/logout Sai da conta
/exit Fecha o Claude
>
i
Dica: os mais usados no dia a dia são /clear (recomeçar do zero), /compact (em conversas longas) e /init (na primeira vez que abre um projeto).

## REGRAS DE COPY

Estas regras separam um carrossel amador de um profissional. Cada uma existe por um motivo — entenda o porquê para aplicar com inteligência:

1. **Cada slide é uma frase de impacto** — O Instagram é visual. Se o slide parece um parágrafo de artigo, ninguém lê. Uma ideia por slide, tipografia gigante. O texto tem que ser legível numa tela de celular a 30cm do rosto.

2. **Dados ESPECÍFICOS** — Nunca "muitos profissionais". Sempre "73% dos profissionais do setor" (Fonte: nome, ano). Número + fonte = credibilidade instantânea. Sem dados, é opinião. Com dados, é autoridade.

3. **Palavras em cor accent** surgem naturalmente no meio do texto — Não em blocos separados. A palavra-chave colorida guia o olho e cria hierarquia visual sem precisar de bullets.

4. **Tom jornalístico** — Como se fosse uma matéria do Estadão ou InfoMoney, não um post de coach. Zero "acredite no seu potencial". Zero "não desista". Informação > inspiração.

5. **Progressão narrativa** — Cada slide puxa pro próximo. O leitor tem que sentir que se parar no slide 3, perdeu o melhor. Ponto sem retorno nos slides 2-3.

6. **Último slide vende sem vender** — Keyword CTA que entrega valor (guia, resumo, checklist, material exclusivo). A pessoa comenta uma palavra e recebe algo. Isso alimenta o algoritmo E gera leads.

7. **Quem compartilha PARECE inteligente** — Esse é o teste final. Se o seguidor compartilhar nos stories ou mandar no WhatsApp, ele parece bem informado? Se sim, o conteúdo está no nível certo.

# Gerador de Propostas Comerciais que Fecham

Você é um especialista em fechamento de vendas com mais de 10 anos de experiência criando propostas que realmente convertem.

## Princípios Fundamentais

- Uma boa proposta **não explica** o que você faz. Ela **prova** o que o cliente vai ganhar.
- A proposta deve ser fácil de entender em 60 segundos ou menos.
- O foco nunca é em você. É sempre no cliente e nos resultados dele.
- Propostas vencedoras reduzem risco percebido e aumentam clareza.

## Estrutura Obrigatória (use sempre nesta ordem)

1. **Resumo Executivo** (máximo 1 página)
   - Entendimento do problema
   - Resultado que ele vai obter
   - Investimento
   - Prazo

2. **Diagnóstico da Situação Atual**
   - Mostre que você entendeu profundamente o contexto dele
   - Use as palavras e dores que ele mencionou

3. **A Solução Proposta**
   - O que exatamente vai entregar
   - Como isso resolve o problema dele

4. **Resultados Esperados**
   - Métricas e resultados realistas
   - Casos parecidos (se tiver)

5. **Escopo + Entregáveis**
   - O que está incluído (muito específico)
   - O que **não** está incluído (evita problemas depois)

6. **Cronograma**
   - Fases + entregas parciais + pontos de aprovação

7. **Investimento**
   - Valor total + formas de pagamento

8. **Próximos Passos**
   - O que ele precisa fazer para começar
   - Validade da proposta

Ato 1 · Conexão Emocional

Os 3 primeiros blocos da página: capturar atenção, ativar dores e criar identificação. Se o Ato 1 falhar, ninguém chega ao resto da página.

Bloco 1 de 3 · Bloco 1 da página

Headline + Badge + CTA

🎯
Objetivo: capturar atenção e comunicar a promessa principal nos primeiros 3 segundos.
1
Badge / Selo

Crie urgência ou exclusividade. Ex: "VAGAS LIMITADAS", "FASE 1", "OFERTA EXCLUSIVA".

2
Headline principal

Fórmula: [Resultado Desejado] + [Sem Objeção Principal] + [Timeframe].

3
Subheadline

Quebre mais objeções e adicione especificidade. Ex: "Usando apenas 20 minutos por dia com seu celular".

4
CTA primário

Botão grande e destacado, com ação clara. Ex: "QUERO ACESSO IMEDIATO".

💡
Exemplo de headline: "Ganhe R$ 5 a 15 Mil/Mês Com TikTok Sem Aparecer e Sem Estoque".
Bloco 2 de 3 · Bloco 2 da página

Identificação com o Problema

🎯
Objetivo: ativar as dores emocionais e criar identificação profunda.
❓

Título provocativo

Pergunta que força reflexão. Ex: "ATÉ QUANDO VOCÊ VAI [DOR PRINCIPAL]?"

📝

Lista de dores

5 a 7 dores específicas, cada uma começando com "Você...", com metáforas visuais fortes.

🖼️

Metáforas

"Sardinha enlatada" (transporte), "peça substituível" (CLT). Linguagem sensorial e emocional.

🪞

Situações reconhecíveis

Cenas específicas do dia a dia que fazem o leitor pensar "isso sou eu".

Bloco 3 de 3 · Bloco 3 da página

História Pessoal (Jornada do Herói)

🎯
Objetivo: criar conexão emocional e estabelecer autoridade através da vulnerabilidade.
1
Apresentação

Nome + credencial atual (resultado alcançado).

2
Situação inicial + Problema

Como era a vida antes, identificação com o público e o que deu errado.

3
Tentativas e falhas

Quantas vezes tentou e o que não funcionou.

4
Fundo do poço

História específica e emocional. Ex: "Pedir o cartão da mãe para comprar arroz".

5
Virada + Descoberta

A decisão tomada e como encontrou o método.

6
Transformação + Missão

Resultados alcançados, vida atual e por que decidiu ajudar outros — a transição para o produto.

Elementos visuais: foto de "antes" (época difícil), foto de "depois" (sucesso atual) e foto com autoridades, se houver.
Aula 03 · Tópico 05

Ato 2 · Construção de Valor

Blocos 4 a 11 da página: apresentar a solução, validar com provas, diferenciar do mercado e empilhar valor até a oferta ficar irresistível.

Bloco 1 de 8 · Bloco 4 da página

Apresentação da Solução

🎯
Objetivo: nomear e posicionar o produto como a solução perfeita.
1
Nome do produto em destaque

Nome memorável, com destaque visual.

2
Descrição em uma frase

O que ele é + o que ele faz. Ex: "A única comunidade que ensina [método] para [resultado]".

3
Qualificações "mesmo que..."

"Mesmo que você nunca…", "Mesmo que você não saiba…", "Mesmo que você já tenha tentado…".

4
Autoridade + transição

"Aqui eu coloquei TODO o conhecimento que…" e feche com "Mas não acredite apenas no que eu estou falando…".

Bloco 2 de 8 · Bloco 5 da página

Prova Social (Depoimentos)

🎯
Objetivo: validar a solução com resultados reais de pessoas comuns.

Título sugerido: "Veja o que pessoas comuns como você estão falando…". Use 3 a 6 depoimentos iniciais, em formatos variados (texto, vídeo, print).

Cada depoimento precisa ter

👤

Nome + foto/vídeo

Pessoa real, identificável, com rosto.

📈

Resultado específico

Número mensurável, não elogio genérico.

⏱️

Timeframe

Quanto tempo levou para alcançar.

🪞

Identificação

Situação "antes" parecida com a do leitor — pessoas comuns, não influencers.

Bloco 3 de 8 · Bloco 6 da página

Diferenciação

🎯
Objetivo: mostrar que o produto é diferente de tudo que existe no mercado.

Título sugerido: "[Produto] é diferente de tudo que você já viu!"

O que NÃO é

Liste 3 a 5 coisas

  • "Não é curso de afiliado comum"
  • "Não é dropshipping tradicional"
  • "Não é mais do mesmo…"

O que É

Posicionamento único

  • Oportunidade única
  • Conectada ao momento de mercado
  • Com validação institucional, parcerias e reconhecimento
Bloco 4 de 8 · Bloco 7 da página

Razões para Funcionar

🎯
Objetivo: quebrar objeções lógicas e explicar por que funciona.

Título sugerido: "[Produto] funciona para você porque…" seguido de 5 a 8 razões específicas, cada uma quebrando uma objeção.

⚙️
Técnica: misture razões emocionais e lógicas, use números sempre que possível e conecte cada razão ao mecanismo único.
Bloco 5 de 8 · Bloco 8 da página

Resumo do Conteúdo

🎯
Objetivo: dar uma visão geral do que está incluído.

Título sugerido: "Em outras palavras… dentro de [Produto] você terá:" com 5 a 8 itens em lista de checkmarks (✅), frases curtas e foco em benefícios, não em recursos.

↔️
Aqui também funciona o contraste antes vs depois: método tradicional (dores) de um lado, seu método (benefícios) do outro.
Bloco 6 de 8 · Bloco 9 da página

Processo Simplificado (3-4 Passos)

🎯
Objetivo: mostrar que é simples e alcançável, reduzindo a percepção de complexidade.

Título sugerido: "De forma simplificada, você terá resultados em apenas [X] passos:"

Cada passo deve ter

🔢

Número + título

Título com verbo de ação. Progressão lógica entre os passos.

✍️

Descrição + ícone

1 a 2 frases específicas mas simples, com ícone ou imagem representativa.

Bloco 7 de 8 · Bloco 10 da página

Qualificação do Público

🎯
Objetivo: fazer o leitor se auto-qualificar e aumentar o desejo (medo de não se qualificar).

Título sugerido: "Até aqui fez sentido, mas… será que [Produto] é pra você?"

✅ Para quem É

5 a 7 perfis

Lista com checkmarks, cada item começando com "Você…".

❌ Para quem NÃO é

3 a 4 perfis

Seja direto e honesto, a qualificação reversa cria desejo e a honestidade gera confiança.

Frase de impacto: "Se você não se encaixa, por favor, feche essa página agora."
Bloco 8 de 8 · Bloco 11 da página

Conteúdo Detalhado (Módulos)

🎯
Objetivo: mostrar transparência e aumentar a percepção de valor.

Título sugerido: "Veja só tudo que você terá acesso dentro de [Produto]:"

🎥

Vídeo tour

Mostre a interface / área de membros e dê uma prévia real do conteúdo.

📚

Lista de módulos

6 a 10 módulos: nome, o que ensina, resultado que entrega e duração (se relevante). 2 a 3 frases cada.

💡
Foque em transformação, não em informação. Ninguém compra aulas, compram o resultado delas.

CTA + confiança

🎯
Objetivo: reforçar credibilidade e criar conexão final.
📸

Foto profissional

Alta qualidade, transmitindo autoridade. Some fotos com autoridades, eventos e prêmios.

🏆

Nome e credenciais

Nome completo, principais títulos e resultados alcançados.

📖

Mini-biografia

Jornada resumida em 3 a 4 parágrafos, com foco em identificação + autoridade.

🔢

Resultados concretos

Números específicos, reconhecimentos e alunos transformados.

🎯

Aula 03 · Tópico 07

Técnicas de Copywriting

As 8 técnicas de escrita que dão vida aos 17 blocos: storytelling, metáforas, números, contraste, perguntas, quebra de padrão, listas e ênfase visual.

Bloco 1 de 4 · Técnica 1

Storytelling

Toda história persuasiva segue 8 etapas: situação inicial → problema → tentativas e falhas → fundo do poço → ponto de virada → descoberta → transformação → missão.

Onde usar

🦸

História principal

Bloco 3 da página, a Jornada do Herói completa.

🎁

Cada bônus

Uma mini-história por bônus. Bônus sem narrativa não gera desejo.

💬

Depoimentos

Antes → depois com contexto emocional.

🔗

Transições

Conectando um bloco ao próximo sem quebra de ritmo.

Bloco 2 de 4 · Técnicas 2 e 3

Metáforas Visuais + Números Específicos

Metáforas poderosas

🖼️
"Sardinha enlatada" (transporte público) · "Peça substituível" (CLT) · "Mapa do tesouro" (lista de produtos) · "GPS personalizado" (diagnóstico) · "Cola dos vídeos" (modelos prontos).
⚙️
Como criar: identifique a sensação → encontre algo visual que a represente → use linguagem sensorial.

Números específicos, sempre

  • ❌ "Muitos seguidores" → ✅ "1,5 milhão de seguidores"
  • ❌ "Vários modelos" → ✅ "50+ modelos de vídeos"
  • ❌ "Investimento alto" → ✅ "Mais de R$ 50 mil investidos"

Use em resultados, valores, quantidades e timeframes.

Bloco 3 de 4 · Técnicas 4 e 5

Contraste + Perguntas Retóricas

Contraste (antes vs depois)

Antes

Método tradicional

  • Dor 1
  • Dor 2
  • Dor 3

Depois

Seu método

  • Benefício 1
  • Benefício 2
  • Benefício 3

Perguntas retóricas

❓
"Você sabe qual é o maior medo de…?" · "Sabe o que isso significa?" · "Você consegue perceber que…?" — engajam o leitor, criam antecipação e forçam reflexão.
Bloco 4 de 4 · Técnicas 6, 7 e 8

Quebra de Padrão, Listas e Ênfase

Quebra de padrão

Frases curtas após parágrafos longos, uma palavra sozinha numa linha, negação seguida de afirmação:

Exemplo de quebra de padrão 
A maioria das pessoas acha que é sorte.

Mentira.

Viralização é ESTRATÉGIA.

Listas e bullets

✅
Use em benefícios, dores, módulos, bônus e FAQ. Símbolos (✅ ❌ ➜), paralelismo e início com verbo ou "Você…".

Ênfase visual

🔤
Negrito para conceitos-chave, CAIXA ALTA para ênfase extrema, itálico para sutileza, riscado para contraste de preço. Regra: use com moderação ou perde o efeito.

Gatilhos Mentais

Os 10 gatilhos que ativam a decisão de compra, como usar cada um e onde aplicar na página.

Bloco 1 de 2 · Gatilhos 1 a 5

Escassez, Urgência, Autoridade, Prova Social e Reciprocidade

⏳

1. Escassez

Como: vagas limitadas, bônus por tempo limitado, preço promocional. Onde: badge no topo, bloco final e próximo aos CTAs.

🚨

2. Urgência

Como: momento de mercado (Fase 1), oportunidade passageira, "essa página pode sair do ar". Onde: diferenciação e bloco final.

🏆

3. Autoridade

Como: resultados alcançados, reconhecimentos, parcerias e entrevistas. Onde: história, "sobre o mentor" e ao longo da página.

👥

4. Prova Social

Como: depoimentos, número de alunos, resultados de terceiros. Onde: bloco 5, ao longo da página e antes de CTAs importantes.

🎁

5. Reciprocidade

Como: muitos bônus, conteúdo de valor na própria página, garantia generosa. Onde: blocos de bônus e garantia.

Coerência, Aversão à Perda, Simplicidade, Identificação e Contraste

🤝

6. Compromisso e Coerência

Como: qualificação ("É pra você?"), perguntas ao longo da página, "se você chegou até aqui…". Onde: qualificação, transições e final.

📉

7. Aversão à Perda

Como: mostrar o que ele perde se não comprar, garantia (não perde nada), "última chance". Onde: garantia e a opção negativa do bloco final.

✨

Como: "em apenas X passos", "usando só 20 minutos", processo simplificado. Onde: headline, descrição do produto e bloco do processo.

🪞

9. Identificação

Como: "pessoas comuns como você", história do mentor, dores específicas. Onde: problema, história e depoimentos.

⚖️

10. Contraste

Como: âncora de preço, antes vs depois, valor total vs preço real. Onde: comparação e oferta.

Estrutura de Bônus

A anatomia do bônus perfeito: nome atrativo, valor destacado, storytelling obrigatório e a objeção que ele quebra.

Bloco 1 de 3 · Anatomia

Anatomia de um bônus perfeito

1
Nome atrativo

Palavras poderosas, específicas, que criam curiosidade. ✅ "Arsenal Viral", "IA Viral Generator", "Plano PIX Imediato" · ❌ "Modelos de vídeos", "Ferramenta de IA".

2
Valor destacado

Sempre atribua um valor em R$, realista mas generoso, com destaque visual.

3
Storytelling (essencial)

Dor → por que você criou → o que tem dentro → comparação poderosa → valor. Detalhado no próximo bloco.

4
Objeção quebrada

Deixe explícito qual objeção o bônus resolve: "não sei criar vídeos", "não tenho tempo", "não sei quais produtos escolher", "não tenho dinheiro para investir".

Bloco 2 de 3 · Storytelling

Storytelling em 5 parágrafos

Aula 03 · Tópico 10

Garantias

A garantia certa reverte 100% do risco e remove a última barreira. Conheça os 4 tipos, os textos modelo e os 6 elementos de uma garantia poderosa.

Bloco 1 de 3 · Tipos

Os 4 tipos de garantia

📅

1. Básica (7-30 dias)

Incondicional, baseada no CDC, sem perguntas. O mínimo obrigatório.

📈

2. De Resultado (60-90 dias)

Condicional: precisa aplicar e provar. Baseada em resultado mínimo. Mais poderosa.

🛡️

3. Dupla (recomendada)

Combina as duas camadas e remove completamente o risco. A mais persuasiva.

⚡

4. Inversa

"Se não funcionar, além de devolver, eu faço um PIX de R$ X para você." Extremamente poderosa, use com cuidado.

🛡️
Garantia dupla: Camada 1 = básica ([X] dias) + Camada 2 = de resultado ([Y] dias) + bônus adicional se não der certo.

1
Período claro

Específico (7, 30, 60 dias) e destacado visualmente.

2
Condições transparentes

3
Processo simples

4
Reversão de risco

"O risco é todo meu", "você não tem nada a perder".

6
Dois cenários positivos

Bloco 1 de 4 · Métricas

5 métricas para acompanhar

🎯

Taxa de conversão

Visitantes → compradores. Meta: 2-5% (tráfego frio) / 5-15% (tráfego quente).

⏱️

Tempo na página

Meta: 5 a 10 minutos no mínimo.

📜

Scroll depth

Até onde rolam. Meta: 70%+ chegam ao final.

👆

Cliques nos CTAs

Meta: 10-20% de taxa de clique.

🛒

Abandono no checkout

Quantos desistem no pagamento. Meta: abaixo de 30%.

Bloco 2 de 4 · Testes

Testes A/B e otimizações rápidas

O que testar

🧪
Headline (promessa, com/sem objeção, ângulos) · Preço (âncora, parcelamento, ordem) · CTAs (texto, cor, posição) · Garantia (período, tipo, posicionamento) · Bônus (quantidade, ordem, valores).

5 otimizações rápidas

1
Adicione mais prova social

Mais depoimentos, número de alunos, resultados específicos.

2
Fortaleça a garantia

Aumente o período, adicione camada de resultado, torne-a mais visual.

3
Simplifique o processo

Reduza para 3-4 passos, mais ícones, linguagem mais simples.

4
Aumente a urgência

Contador, escassez em destaque, timing reforçado.

5
Melhore os CTAs

Texto mais específico, cor mais contrastante, elementos de confiança.

Bloco 3 de 4 · Erros

10 erros que matam conversão

  • ❌ Tentar convencer de muitas coisas — foque na Única Crença
  • ❌ Bônus sem storytelling — cada bônus precisa de narrativa
  • ❌ Depoimentos genéricos — foque em resultados específicos
  • ❌ Garantia fraca — seja generoso e claro
  • ❌ Falta de urgência — sempre tenha elemento de timing
  • ❌ Linguagem corporativa — escreva como você fala
  • ❌ Falta de prova social — nunca é demais
  • ❌ Preço sem âncora — sempre use contraste
  • ❌ FAQ que não quebra objeções — use perguntas reais
  • ❌ CTA fraco — seja específico e direto
