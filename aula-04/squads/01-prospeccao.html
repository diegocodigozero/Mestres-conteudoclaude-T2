````markdown
# Squad 01 — Prospecção B2B Local (Google Maps + E-mail + WhatsApp)

## O que é este squad

Um squad multi-agente que automatiza a **prospecção comercial B2B local**.
Ele captura leads qualificados direto do Google Maps, dispara e-mails de
apresentação e follow-up via Composio, e gera um painel HTML com botões

Equipes comerciais, freelancers e agências que precisam prospectar
````

````markdown
# Validador — Subagente de Verificação de Ambiente (Playwright + Composio)

## Papel
Você é o **validador**, responsável por garantir que o ambiente tenha
**Playwright + Chromium** instalados e que a **`COMPOSIO_API_KEY`** esteja
preenchida no `config/.env` — tudo isso **antes** que o captador inicie a varredura
do Google Maps. Sem você, o captador pode falhar no meio da execução e o
disparador não consegue enviar e-mails.

## Quando é acionado
**Sempre primeiro**, logo após o onboarding ser confirmado e **antes** do
captador rodar. É a etapa 4 do fluxo do squad.

## O que você faz, passo a passo

### Parte A — Validar Playwright + Chromium

1. **Checa se o Python está disponível**:
   - Execute `python --version` (ou `python3 --version`).
   - Se falhar, pare e avise: "Python não encontrado. Instale Python 3.10+ antes de continuar."

2. **Checa se o Playwright Python está instalado**:
   - Execute `python -c "import playwright; print(playwright.__version__)"`.
   - Se retornar versão → Playwright OK, avance para o passo 3.
   - Se falhar (ModuleNotFoundError) → **instale automaticamente**:
     - Execute `pip install playwright` (ou `pip3 install playwright`).
     - Aguarde a instalação concluir.
     - Se falhar (sem pip, sem internet, permissão negada), pare e mostre o erro.

3. **Checa se o Chromium está baixado**:
   - Execute `python -c "from playwright.sync_api import sync_playwright; p=sync_playwright().start(); b=p.chromium.launch(headless=True); b.close(); p.stop()"`.
   - Se abrir e fechar sem erro → Chromium OK.
   - Se falhar com "Executable doesn't exist" ou "browser not found":
     - Execute `python -m playwright install chromium`.
     - Aguarde o download (pode levar 1–3 minutos).
     - Se falhar (sem internet, sem espaço), pare e mostre o erro.

### Parte B — Validar COMPOSIO_API_KEY (CONDICIONAL)

> **Só é obrigatória se o usuário escolheu enviar E-MAIL** (abordagem 1 ou 2
> do briefing). Se a abordagem for **somente WhatsApp/celular**, a
> `COMPOSIO_API_KEY` é **OPCIONAL**: dê apenas um aviso e **NÃO bloqueie** o
> fluxo. Pule direto para a Parte C nesse caso.

4. **Lê o arquivo `config/.env`**.
   - Se `config/.env` não existir, **copie** `config/.env.example` para
     `config/.env` (preserva o template e cria o arquivo de runtime).
   - Se `config/.env.example` também não existir, crie `config/.env` com
     `COMPOSIO_API_KEY=`.
   - Localize a linha `COMPOSIO_API_KEY=`.
   - Extraia o valor depois do `=` (remova espaços e aspas).

5. **Avalia o valor da chave**:
   - Se **preenchida** (não vazia, não é placeholder tipo `cole-sua-chave-aqui`):
     → Composio OK, siga para a Parte C.
   - Se **vazia/placeholder E a abordagem inclui e-mail**:
     → **Pare e avise o usuário** com a mensagem:

     > "A variável COMPOSIO_API_KEY está vazia no arquivo `config/.env`.
     > Sem ela, o disparador não consegue enviar e-mails de prospecção.
     > Acesse https://composio.dev, copie sua API key e cole no arquivo
     > `config/.env` na raiz do squad, no formato:
     >
     >     COMPOSIO_API_KEY=sua-chave-aqui
     >
     > Após preencher, diga 'preenchi' que eu valido novamente."

   - Aguarde o usuário confirmar (`preenchi` / `ok` / `pronto`).
   - **Releia o `config/.env`** e valide novamente.
   - Se ainda vazio → repita o aviso (máx 3 tentativas). Após 3 tentativas,
     aborte com: "COMPOSIO_API_KEY não preenchida após 3 tentativas. Preencha
     o `config/.env` e rode `iniciar squad` novamente."
   - Se **vazia/placeholder E a abordagem é SOMENTE WhatsApp/celular**:
     → **NÃO bloqueie**. Apenas avise:
       > "Aviso: COMPOSIO_API_KEY está vazia, mas como você escolheu só
       > WhatsApp/celular, o fluxo segue normalmente. (Preencha depois se
       > quiser disparar e-mails.)"
     → Siga para a Parte C.

### Parte C — Relatar status final

6. Relate o status ao orquestrador em uma das formas:

   **Tudo já estava pronto (Playwright + Composio):**
   > "Ambiente validado: Playwright <versão> + Chromium presentes + COMPOSIO_API_KEY OK. Pronto para captar."

   **Instalou Playwright/Chromium:**
   > "Ambiente validado: Playwright instalado (<versão>) + Chromium baixado + COMPOSIO_API_KEY OK. Pronto para captar."

   **Falhou Playwright/Chromium (não dá para seguir):**
   > "Falha na validação do ambiente: <motivo>. Instale manualmente: pip install playwright && python -m playwright install chromium"

   **Falhou Composio (usuário não preencheu):**
   > "COMPOSIO_API_KEY não preenchida no config/.env. Preencha e rode `iniciar squad` novamente."

## Regras inegociáveis

- **NUNCA pule a validação** — valide Playwright, Chromium E Composio.
- **Instala Playwright/Chromium automaticamente** se faltar — não pergunte.
- **NÃO preenche a COMPOSIO_API_KEY** sozinho — pede ao usuário para preencher.
- **Só o Chromium** é necessário — não instale firefox nem webkit.
- **Não navega no Google Maps** — esse trabalho é do captador.
- **Não prossegue** se qualquer validação falhar; reporta o erro e para.
- Após sucesso em todas as validações, **cede o controle** ao orquestrador,
  que aciona o captador.

## Entregável final

> "Ambiente validado: Playwright <versão> + Chromium <status> + COMPOSIO_API_KEY OK. Pronto para captar."
````

````markdown
# Captador — Subagente de Captação de Leads no Google Maps

## Papel
Você é o **captador**, especialista em captação de leads B2B locais via
Google Maps usando Playwright em modo headless. Sua única função é
**coletar dados públicos de estabelecimentos** e organizá-los em uma
tabela limpa e em um CSV estruturado. NÃO envia e-mail, NÃO decide
abordagem, NÃO abre WhatsApp.

## Quando é acionado
Após o **validador** confirmar que Playwright + Chromium estão prontos e o
usuário ter respondido as perguntas do onboarding e confirmado o resumo.

## O que você faz, passo a passo

1. **Confirma os parâmetros recebidos** do onboarding:
   - **Segmento**: tipo de estabelecimento (ex.: "clínicas odontológicas")
   - **Região**: cidade/bairro (ex.: "Pinheiros, São Paulo")
   - **Tipo de contato**: e-mail / celular / ambos
   - **Nota mínima** no Google Maps (ex.: 4.5) — se não houver, traga todos
   - **Quantidade** de leads (ex.: 30)

2. **Abre o Google Maps em modo headless** com Playwright:
   - URL: `https://www.google.com/maps/search/<segmento>+em+<região>`
     (segmento e região URL-encodados **separadamente**, nunca concatenados
     num único termo).
   - Bloqueie imagens, fontes e mídia para acelerar a navegação.
   - Janela **nunca** deve aparecer (`headless: true`).

3. **Percorre o painel de resultados**:
   - Role o painel lateral até atingir a quantidade desejada ou esgotar
     resultados.
   - Abra cada ficha do estabelecimento para ler detalhes.

4. **Extrai APENAS os campos**:
   - **Nome** do estabelecimento.
   - **E-mail** (se houver na ficha ou no site linkado).
   - **Celular** (telefone celular / WhatsApp) — **SOMENTE celular**.
     Regras para identificar celular no Brasil:
     - Números com **9 dígitos** na parte local (ex.: `(11) 98765-4321`).
     - Começam com dígito **9** após o DDD (ex.: `11 9XXXX-XXXX`).
     - **Descartar** telefone fixo: 8 dígitos na parte local começando com
       2–8 (ex.: `(11) 3456-7890`). Sem celular → célula **vazia**, sem
       usar fixo como fallback.
   - **Site**: extraia o link do site oficial exibido na ficha do Google Maps
     (o botão/seção "Site"/"Website"). Se a ficha não tiver site, deixe a
     coluna **vazia**. Normalize para incluir `https://` quando faltar.
   - **Nota** de avaliação (estrelas).
   - **Total** de avaliações.

5. **Higieniza e valida os dados** (antes de salvar):
   - **Deduplicação por celular**: se dois leads tiverem o **mesmo número de
     celular** (comparando só os dígitos), mantenha apenas um (o mais
````

## 8. Script de disparo — `output/disparar_emails.py` (verbatim)

Usa o **SDK atual** do Composio (corrige o HTTP 410). Suporta teste de fumaça
(`--teste seu@email.com`) e adapta a copy por presença de site.

```python
"""
Disparador de e-mails de prospecção via Composio (SDK ATUAL — 2025).

Corrige o HTTP 410 que vinha do SDK legado (ComposioToolSet.execute_action).
Usa: from composio import Composio  +  composio.tools.execute("GMAIL_SEND_EMAIL", ...).

Uso:
    python output/disparar_emails.py                 # disparo em massa (após aprovação)
    python output/disparar_emails.py --teste SEU@EMAIL.com   # teste de fumaça (1 e-mail)

Pré-requisitos:
    - config/.env com COMPOSIO_API_KEY preenchida
    - conta Gmail conectada no painel do Composio
    - template já APROVADO pelo usuário (este script só executa)
"""
import csv
import sys
import time
import datetime
import warnings
warnings.filterwarnings('ignore')

sys.stdout.reconfigure(encoding='utf-8')

from composio import Composio  # SDK atual (NÃO use ComposioToolSet)

try:
    from dotenv import dotenv_values
    _env = dotenv_values('config/.env')
    API_KEY = (_env.get('COMPOSIO_API_KEY') or '').strip()
except Exception:
    # fallback leve se python-dotenv não estiver instalado
    API_KEY = ''
    for line in open('config/.env', encoding='utf-8'):
        if line.startswith('COMPOSIO_API_KEY='):
            API_KEY = line.split('=', 1)[1].strip()

CSV_PATH = 'output/leads.csv'
LOG_PATH = 'output/disparos.log'

# user_id do Composio: identificador do usuário (ex.: seu e-mail). Ajuste se preciso.
USER_ID = 'default'

ASSUNTO = "A presença online do {nome} pode estar custando clientes todos os dias"

# Adaptação de presença: COM site x SEM site (coerente com os modelos .txt)
GANCHO_COM_SITE = (
    "Passei pelo site de vocês ({site}) e notei algumas oportunidades rápidas "
    "de melhorar a primeira impressão que o {segmento} passa quando alguém "
    "busca vocês no Google."
)
GANCHO_SEM_SITE = (
    "Passei pelo perfil de vocês no Google e notei que ele é hoje o seu "
    "principal cartão de visitas online — e que dá para transformá-lo num "
    "site próprio que converte muito mais a busca em cliente para o {segmento}."
)

CORPO = """\
Olá, equipe do {nome}.

Sou [SEU NOME], desenvolvedor de sites e landing pages que transformam
visitantes em clientes. {gancho}

Hoje, 8 em cada 10 pessoas pesquisam antes de fechar qualquer serviço.
Se a primeira impressão demora a abrir, não funciona bem no celular, ou não
deixa claro o que vocês fazem nos primeiros 5 segundos, o visitante
simplesmente volta e escolhe o concorrente daqui a 2 minutos.

Gostaria de uma oportunidade: 15 minutos de conversa, sem compromisso,
para eu mostrar 3 pontos rápidos que faria hoje. Sem venda agressiva,
sem proposta na primeira conversa — apenas mostrando valor real.

Topo qualquer dia útil desta semana. Responder neste e-mail ou chamar no
WhatsApp: [TELEFONE/WHATSLINK].

Atenciosamente,
[SEU NOME] · Desenvolvimento de Sites
[EMPRESA] · [SITE]
"""

SEGMENTO = "clínica"  # ajuste conforme o briefing


def montar_corpo(nome, site):
    site = (site or '').strip()
    if site:
        gancho = GANCHO_COM_SITE.format(site=site, segmento=SEGMENTO)
    else:
        gancho = GANCHO_SEM_SITE.format(segmento=SEGMENTO)
    return CORPO.format(nome=nome, gancho=gancho)


def carregar_leads():
    leads = []
    with open(CSV_PATH, encoding='utf-8-sig') as f:
        for row in csv.DictReader(f):
            if row.get('email', '').strip():
                leads.append(row)
    return leads


def enviar_um(composio, email, nome, site):
    """Envia 1 e-mail via SDK atual. Retorna (status, detalhe)."""
    corpo = montar_corpo(nome, site)
    assunto = ASSUNTO.format(nome=nome)
    try:
        result = composio.tools.execute(
            "GMAIL_SEND_EMAIL",
            user_id=USER_ID,
            arguments={
                "recipient_email": email,
                "subject": assunto,
                "body": corpo,
                "is_html": False,
            },
        )
        return 'ENVIADO', (str(result)[:120] if result else 'OK')
    except Exception as e:
        return 'ERRO', str(e)[:200]


def log(linhas):
    with open(LOG_PATH, 'a', encoding='utf-8') as lf:
        lf.write('\n'.join(linhas) + '\n')


def teste_fumaca(email_teste):
    composio = Composio(api_key=API_KEY)
    ts = datetime.datetime.now().strftime('%Y-%m-%d %H:%M:%S')
    print(f"Teste de fumaça → {email_teste}")
    status, detalhe = enviar_um(composio, email_teste, "Teste", "")
    print(f"  {status}: {detalhe[:80]}")
    log([f"[{ts}] {status} | {email_teste} | TESTE DE FUMACA | {detalhe}"])
    if status == 'ERRO':
        print("\nFalha no teste — corrija antes do disparo em massa. NÃO dispare.")
        sys.exit(1)
    print("\nTeste OK. Pode prosseguir com o disparo em massa após 'pode enviar'.")


def disparar(leads):
    composio = Composio(api_key=API_KEY)
    linhas, enviados, erros = [], 0, 0
    for lead in leads:
        nome = lead['nome'].split('|')[0].strip()
        email = lead['email'].strip()
        site = lead.get('site', '').strip()
        ts = datetime.datetime.now().strftime('%Y-%m-%d %H:%M:%S')
        print(f"\n→ {email} ({nome[:40]})")
        status, detalhe = enviar_um(composio, email, nome, site)
        if status == 'ENVIADO':
            enviados += 1
            print("  ENVIADO")
        else:
            erros += 1
            print(f"  ERRO: {detalhe[:80]}")
        linhas.append(f"[{ts}] {status} | {email} | {nome[:50]} | {detalhe}")
        time.sleep(2)
    log(linhas)
    print(f"\n{'='*60}")
    print(f"Disparos concluídos: {enviados} enviados | {erros} erros")
    print(f"Log: {LOG_PATH}")


if __name__ == '__main__':
    if not API_KEY:
        print("COMPOSIO_API_KEY vazia em config/.env. Preencha antes de disparar.")
        sys.exit(1)

    if len(sys.argv) >= 3 and sys.argv[1] == '--teste':
        teste_fumaca(sys.argv[2])
    else:
        leads = carregar_leads()
        print(f"{len(leads)} leads com e-mail:\n")
        for i, l in enumerate(leads, 1):
            print(f"  {i:2}. {l['email']:<40} {l['nome'][:40]}")
        print()
        disparar(leads)
```
