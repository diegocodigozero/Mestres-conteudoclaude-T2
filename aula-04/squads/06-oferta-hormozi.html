## Os ângulos

No **modo demo** o `maestro` define apenas o **Ângulo A**. No **modo completo**
define 3 ângulos distintos após o onboarding. Estrutura default:

| Ângulo | Foco | Avatar | Promessa | Preço |
|--------|------|--------|----------|-------|
| **A — Premium** | Alto ticket, baixo volume, consultivo | Decisor com dor cara | Transformação completa | $$$ |
| **B — Volume** | Ticket médio, escala, self-serve | Iniciante com dor frequente | Resultado rápido | $$ |
| **C — Acessível** | Baixo ticket, alto volume, impulso | Massa com dor barata | Quick win imediato | $ |

A nomeação real de cada ângulo vem do agente `naming-hook` de cada pipeline.

```

  → briefing.md

  → OS ÂNGULOS RODAM EM PARALELO entre si (no modo completo);
    DENTRO de cada ângulo a cadeia é SEQUENCIAL (12 agentes em cadeia):
      avatar-hunter → dor-bleeding-neck → veiculo-solucao →
      equacao-valor → bonus-stacker → scarcity-urgency →
      naming-hook → pricing-architect → funnel-architect →
      traffic-strategist → sales-closer → fulfillment-ops
      → PLANO-MESTRE-<X>.md consolidado
  → consolidador-html
      → plano-mestre.html

```

**Paralelo vs sequencial:** os **ângulos** rodam em **paralelo** entre si
(modo completo); mas **dentro de cada ângulo** os 12 agentes rodam em
**cadeia sequencial** — cada agente só inicia quando o entregável do anterior
existir na pasta do ângulo. No modo demo há apenas o ângulo A (uma cadeia).

## Onboarding (gatilhos: `iniciar squad demo` / `iniciar squad`)

1. **Banner ASCII** de boas-vindas + 2 linhas (quem é + o que faz).
2. **Passo 0 — Pré-requisitos** (validar de verdade ANTES de perguntar):
   - `[OK]/[FALTA]` pasta `output/` gravável
   - `[OK]/[FALTA]` conexão à internet (CDN do Mermaid para os diagramas)
   - Este squad **não usa API key/secret** — não há `config/.env`.
3. **Fazer as 8 perguntas DE UMA VEZ** (um único bloco, com exemplos entre
   parênteses). No modo demo, oferecer "usar exemplo" (briefing pronto de
   "curso de inglês para devs") para o aluno rodar sem ter produto:

   ```
   Responda as 8 perguntas de uma vez (ou "usar exemplo" para o demo):

   1) Produto/serviço (ex.: "curso de inglês para devs")
   2) Modalidade: info, SaaS, serviço, físico, híbrido? (ex.: "info")
   3) Preço atual (ex.: "R$ 497" ou "não vendo ainda")
   4) Avatar atual (ex.: "dev pleno" ou "não sei ainda")
   5) Ticket médio desejado (ex.: "R$ 1.997")
   6) Budget mensal de tráfego (ex.: "R$ 3.000/mês")
   7) Canal de venda hoje: DM, call, página, loja física, "nenhum"? (ex.: "página")
   8) Idioma de saída: pt-BR, en, es? (ex.: "pt-BR")
   ```

4. **Confirmar resumo** do briefing antes de prosseguir (indicar a etapa,
   ex.: "Etapa 2 de 5").
5. O `maestro` gera `briefing.md` em `output/<produto-slug>/`.
6. O `maestro` define os ângulos (demo: só A; completo: A/B/C) e cria as pastas.
7. Dispara a **cadeia sequencial** de 12 agentes em cada ângulo;
   **os ângulos rodam em paralelo entre si** (modo completo).
8. Ao término, aciona o `consolidador-html`.
9. Apresenta o caminho do `plano-mestre.html` final ao usuário.

## HTML final — estilo Hormozi

O `consolidador-html` gera `plano-mestre.html` com:

- **Header fixo** laranja (#FF6B00) / preto / bold
- **Abas** — demo: 1 aba (Ângulo A); completo: 4 abas (A / B / C + **Comparativa**)
- **Badge de ratio de valor** (`.value-anchor`) no topo de cada aba de ângulo:
  "Valor ancorado total vs Preço (ratio 10:1)" com os números reais
- **Aba Comparativa** (só no modo completo): tabela A/B/C lado a lado
  (avatar, preço, promessa, ticket, ratio de valor)
- **Sidebar sticky** por aba de ângulo com as 12 seções (scroll suave)
- **Mermaid.js via CDN** para os diagramas de funil. **Precisa de internet**
  na primeira carga; sem rede, o código do funil aparece como fallback.
  O HTML re-dispara `window.mermaid.run()` ao trocar de aba, porque as abas
  ocultas usam `display:none` e não renderizam sozinhas.
- **CSS inline** (sem dependências externas além do Mermaid)
- **Tabelas estilizadas** (avatar, pricing, equação de valor, objeções)
- **Cards** para bônus e garantias
- **Badges coloridos** para escassez/urgência
- **Print-friendly** (media print)
- Cada aba renderiza o conteúdo do `PLANO-MESTRE-<X>.md` convertido em HTML

## Convenções de saída

- Datas no formato `YYYY-MM-DD`.
- `<produto-slug>` é derivado do nome do produto (kebab-case).
- Todos os caminhos referenciados pelos agentes são **relativos** à raiz do squad.
- Os 12 arquivos `.md` de cada ângulo são numerados `01-` a `12-` para ordenação.
- O `PLANO-MESTRE-<X>.md` concatena os 12 arquivos do ângulo em um único documento navegável.
- O `plano-mestre.html` é o entregável final que o usuário abre no browser.

# Skill: Calcular Equação de Valor

## Descrição

Aplicar a **Equação de Valor** de Alex Hormozi para maximizar o valor
percebido da oferta:

```
Valor = (Dream Outcome × Perceived Likelihood of Achievement)
        / (Time Delay × Effort & Sacrifice)
```

O objetivo é subir o numerador (Dream Outcome e Likelihood) e diminuir
o denominador (Time Delay e Effort & Sacrifice).

## Quando acionar

Pelo agente `equacao-valor`, após `veiculo-solucao` entregar `03-veiculo.md`.

## Passos de execução

1. **Ler `03-veiculo.md`** para entender a promessa, o mecanismo e a
   transformação.
2. **Dream Outcome (↑)** — listar 10 outcomes que o cliente sonha alcançar.
   Ir além do resultado funcional: incluir status, reconhecimento,
   liberdade, paz de mente, etc.
3. **Perceived Likelihood of Achievement (↑)** — estratégias para aumentar
   a percepção de que vai funcionar:
   - Prova social (depoimentos, números, cases)
   - Garantia (o que devolve se não funcionar)
   - Resultado parcial garantido (quick win)
   - Autoridade (credenciais, experiência)
4. **Time Delay (↓)** — estratégias para reduzir o tempo até o primeiro
   resultado:
   - Quick win em 24h/7dias
   - Tempo médio até resultado completo
   - Marcos acelerados
5. **Effort & Sacrifice (↓)** — estratégias para reduzir esforço e
   sacrifício:
   - Componentes "done-for-you" (feito pra você)
   - Simplificações (templates, atalhos)
   - O que o cliente NÃO precisa fazer
6. **Calcular o valor percebido**:
   - Somar o valor ancorado de tudo que está incluído (oferta + bônus
     do `05-stack-bonus.md` se disponível, ou estimar)
   - Comparar com o preço de venda
   - Ratio deve ser ≥ 10:1
7. **Gravar** `04-equacao-valor.md` na pasta do ângulo.

## Template do entregável

Ver `equacao-valor.md` → seção "Estrutura do entregável".

## Regras

- Dream Outcome deve ter exatamente 10 itens (nem mais nem menos).
- Cada estratégia de Likelihood deve ser concreta e acionável.
- Time Delay deve incluir um quick win alcançável.
- Effort & Sacrifice deve listar pelo menos 3 coisas que o cliente
  NÃO precisa fazer.
- O ratio valor:preço deve ser ≥ 10:1. Se não for, reportar ao maestro.

### Como rodar

1. Abrir o Claude Code na pasta `06-oferta-hormozi/`.
2. Digitar **`iniciar squad demo`** (recomendado — 1 ângulo, rápido) ou
   **`iniciar squad`** (modo completo — 3 ângulos).
3. O `maestro` mostra o banner, valida o Passo 0 (output/ gravável +
   internet), e faz as 8 perguntas em um único bloco. No modo demo é
   possível responder **"usar exemplo"** (curso de inglês para devs).
4. Confirmar o resumo do briefing.
5. O squad roda a cadeia sequencial de 12 agentes por ângulo (ângulos em
   paralelo no modo completo) e consolida tudo.
6. Abrir o `output/<produto-slug>/plano-mestre.html` no navegador
   (com internet para ver os diagramas Mermaid).

### Notas

- **Sem secrets:** este squad não usa API key; não há `config/.env`.
- **Mermaid exige internet** na 1ª carga do HTML; sem rede, o código do
  funil aparece como fallback.
- **Paralelo vs sequencial:** ângulos em paralelo; dentro de cada ângulo,
  cadeia sequencial.
